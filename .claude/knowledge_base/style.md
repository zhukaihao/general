# 测试用例编写规范文档

## 一、用例结构规范

### 1.1 标题层级

```
##武将名
### 技能名
####技能完整描述
#####关键字短语
###### 细节测试点
####### 预期结果/举例
```

- **固定6层树形结构**：武将名 → 技能名 → 技能完整描述 → 关键字短语 → 细节测试点 → 预期结果/举例
- **第3层必须是父节点**：完整展示技能描述文本，作为后续拆解的父级
- **禁止添加前缀**：不添加「关键字：」「子节点：」等中间层

### 1.2 关键字短语选择

- **使用原句/核心短语**：直接取自技能描述的完整句子，不精简
- **每个关键字独立成节点**：作为第3层父节点的子节点

### 1.3 节点命名原则

- 使用陈述性描述，不含"测试"字样
- 描述预期结果而非操作步骤
- 极简短语，1-10字
- 特殊情况单独成节：使用「特殊情况」标记异常分支


### 1.4 预期结果表达

| 场景 | 表达方式 |
|------|----------|
| **生效/不生效** | 计算在内 / 不计算在内 |
| **数值变化** | 伤害+1 / 伤害+2 / 造成3点伤害 |
| **界面反馈** | 显示icon、点击显示tip、座位上出现icon |
| **交互反馈** | 可以取消发动、超时默认不发动 |
| **限制生效** | 只能选择XX、必须指定XX为目标 |
